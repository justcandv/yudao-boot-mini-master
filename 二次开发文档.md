# 芋道（yudao-boot-mini）二次开发文档

> 基于 yudao-boot-mini-master-jdk17 版本，Spring Boot 3.5.9 + JDK 17

---

## 一、项目概述

芋道（yudao）是一个基于 Spring Boot 的快速开发平台，采用模块化架构设计，提供了完善的权限管理、多租户、代码生成等企业级功能。本项目为其精简版（mini），默认启用 `system` 和 `infra` 两个核心模块。

### 1.1 技术栈总览

| 分类 | 技术 | 版本 |
|------|------|------|
| 核心框架 | Spring Boot | 3.5.9 |
| JDK 版本 | JDK | 17 |
| ORM 框架 | MyBatis-Plus | 3.5.15 |
| 联表查询 | MyBatis-Plus-Join | 1.5.5 |
| 数据库连接池 | Druid | 1.2.27 |
| 多数据源 | Dynamic Datasource | 4.5.0 |
| 缓存 | Redis + Redisson | 3.52.0 |
| 安全框架 | Spring Security | (随 Spring Boot) |
| 接口文档 | Knife4j + SpringDoc | 4.5.0 / 2.8.14 |
| 定时任务 | Quartz | (随 Spring Boot) |
| 消息队列 | Redis/RocketMQ/Kafka/RabbitMQ | 多选一 |
| 对象转换 | MapStruct | 1.6.3 |
| 工具库 | Hutool | 5.8.42 |
| Excel 处理 | FastExcel | 1.3.0 |
| 代码简化 | Lombok | 1.18.42 |
| 分布式锁 | Lock4j | 2.2.7 |
| 模板引擎 | Velocity | 2.4.1（代码生成） |

### 1.2 前端项目

项目包含多套前端方案，位于 `yudao-ui/` 目录下：

| 前端项目 | 说明 |

## 三、核心框架类详解

### 3.1 统一返回结果 `CommonResult<T>`

所有接口统一返回 `CommonResult<T>` 对象：

```java
// 成功返回
CommonResult.success(data)

// 错误返回
CommonResult.error(errorCode)
CommonResult.error(code, message)
```

返回 JSON 格式：
```json
{
  "code": 0,        // 0 表示成功，其他为错误码
  "msg": "",         // 错误信息
  "data": { ... }   // 业务数据
}
```

### 3.2 分页参数与结果

**请求参数** — `PageParam`：
```java
public class PageParam {
    private Integer pageNo = 1;    // 页码，从 1 开始
    private Integer pageSize = 10; // 每页条数，最大 200
}
```

**返回结果** — `PageResult<T>`：
```java
public class PageResult<T> {
    private Long total;      // 总数
    private List<T> list;    // 数据列表
}
```

### 3.3 异常体系

```
ErrorCode                          # 错误码对象（code + msg）
├── GlobalErrorCodeConstants       # 全局错误码（0-999）: SUCCESS(0), BAD_REQUEST(400), UNAUTHORIZED(401)...
└── ServiceErrorCodeRange          # 业务错误码分段规则（10 位数字）

ServiceException                   # 业务异常类（继承 RuntimeException）
ServiceExceptionUtil               # 业务异常工具类，快速抛出异常

GlobalExceptionHandler             # 全局异常处理器（@RestControllerAdvice）
```

**错误码分段规则（10 位）**：

| 段 | 位数 | 说明 | 示例 |
|---|------|------|------|
| 第 1 段 | 1 位 | 类型 | `1` = 业务异常 |
| 第 2 段 | 3 位 | 系统 | `001` = infra, `002` = system |
| 第 3 段 | 3 位 | 模块 | `001` = OAuth2, `002` = User |
| 第 4 段 | 3 位 | 错误序号 | 自增 |

**使用示例**：
```java
// 1. 定义错误码（在模块的 ErrorCodeConstants 中）
ErrorCode USER_NOT_EXISTS = new ErrorCode(1_002_002_001, "用户不存在");

// 2. 抛出业务异常
throw ServiceExceptionUtil.exception(USER_NOT_EXISTS);
```

### 3.4 数据库基础类

#### BaseDO — 基础实体对象

```java
public abstract class BaseDO implements Serializable {
    private LocalDateTime createTime;  // 创建时间（自动填充）
    private LocalDateTime updateTime;  // 更新时间（自动填充）
    private String creator;            // 创建者 ID（自动填充）
    private String updater;            // 更新者 ID（自动填充）
    private Boolean deleted;           // 逻辑删除标记（@TableLogic）
}
```

#### TenantBaseDO — 多租户基础实体

```java
public abstract class TenantBaseDO extends BaseDO {
    private Long tenantId;  // 租户编号（自动填充）
}
```

> **规则**：需要多租户隔离的表继承 `TenantBaseDO`，不需要的继承 `BaseDO`。

#### BaseMapperX<T> — 增强 Mapper

继承自 MyBatis-Plus 的 `BaseMapper`，扩展了以下能力：

```java
public interface BaseMapperX<T> extends MPJBaseMapper<T> {
    // 分页查询
    PageResult<T> selectPage(PageParam pageParam, Wrapper<T> queryWrapper);
    
    // 联表分页查询
    <D> PageResult<D> selectJoinPage(PageParam pageParam, Class<D> clazz, MPJLambdaWrapper<T> wrapper);
    
    // 便捷查询
    T selectOne(SFunction<T, ?> field, Object value);
    List<T> selectList(SFunction<T, ?> field, Object value);
    Long selectCount(SFunction<T, ?> field, Object value);
    
    // 批量操作
    Boolean insertBatch(Collection<T> entities);
    Boolean updateBatch(Collection<T> entities);
}
```

#### LambdaQueryWrapperX<T> — 增强查询条件

```java
// 核心特性：xxxIfPresent 方法，参数为空时自动跳过该条件
new LambdaQueryWrapperX<AdminUserDO>()
    .likeIfPresent(AdminUserDO::getUsername, reqVO.getUsername())   // username 不为空才添加 LIKE
    .eqIfPresent(AdminUserDO::getStatus, reqVO.getStatus())        // status 不为空才添加 =
    .betweenIfPresent(AdminUserDO::getCreateTime, reqVO.getCreateTime()) // 时间范围
    .orderByDesc(AdminUserDO::getId);
```

### 3.5 安全框架

#### LoginUser — 登录用户信息

```java
public class LoginUser {
    private Long id;              // 用户 ID
    private Integer userType;     // 用户类型（1-管理员, 2-会员）
    private Long tenantId;        // 租户 ID
    private List<String> scopes;  // 授权范围
    private LocalDateTime expiresTime; // 过期时间
}
```

#### 权限控制

```java
// Controller 方法上使用 @PreAuthorize 注解
@PreAuthorize("@ss.hasPermission('system:user:create')")
public CommonResult<Long> createUser(...) { ... }

// @ss 引用的是 SecurityFrameworkServiceImpl Bean
// 'system:user:create' 是权限标识，格式：模块:资源:操作
```

#### 获取当前登录用户

```java
// 方式一：通过工具类
Long userId = SecurityFrameworkUtils.getLoginUserId();
LoginUser loginUser = SecurityFrameworkUtils.getLoginUser();

// 方式二：通过 WebFrameworkUtils（在 Controller 中）
Long userId = WebFrameworkUtils.getLoginUserId(request);
```

### 3.6 多租户

- **数据库级隔离**：通过 `TenantDatabaseInterceptor` MyBatis 拦截器，自动在 SQL 中追加 `tenant_id` 条件
- **Redis 级隔离**：通过 `TenantRedisCacheManager` 给 Redis Key 添加租户前缀
- **安全级隔离**：通过 `TenantSecurityWebFilter` 校验请求头中的租户信息
- **忽略租户**：使用 `@TenantIgnore` 注解或 `TenantUtils.executeIgnore()` 工具方法

### 3.7 数据权限

通过 `@DataPermission` 注解控制数据范围：

```java
@DataPermission(enable = true)  // 启用数据权限过滤
public PageResult<AdminUserDO> getUserPage(UserPageReqVO reqVO) { ... }
```

系统支持 5 种数据权限范围：全部、本部门及以下、本部门、仅本人、自定义。

## 四、代码分层规范

### 4.1 请求处理流程

```
HTTP 请求
  ↓
TokenAuthenticationFilter        # Token 认证过滤器
  ↓
TenantContextWebFilter           # 租户上下文设置
  ↓
Controller                       # 接收请求，参数校验
  ↓ 调用
Service（Interface → Impl）      # 业务逻辑
  ↓ 调用
Mapper（BaseMapperX）            # 数据访问
  ↓
数据库 / Redis
```

## 五、新增业务模块指南（完整步骤）

以新增一个 **商品管理模块（product）** 为例：

### 步骤 1：创建数据库表

```sql
CREATE TABLE product (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '商品ID',
    name            VARCHAR(100)  NOT NULL COMMENT '商品名称',
    price           INT           NOT NULL COMMENT '价格（单位：分）',
    status          TINYINT       NOT NULL DEFAULT 0 COMMENT '状态（0-禁用 1-启用）',
    description     VARCHAR(500)  DEFAULT NULL COMMENT '描述',
    tenant_id       BIGINT        NOT NULL DEFAULT 0 COMMENT '租户编号',
    creator         VARCHAR(64)   DEFAULT '' COMMENT '创建者',
    create_time     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updater         VARCHAR(64)   DEFAULT '' COMMENT '更新者',
    update_time     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted         BIT(1)        NOT NULL DEFAULT b'0' COMMENT '是否删除'
) COMMENT = '商品表';
```

> **字段规范**：`tenant_id`、`creator`、`create_time`、`updater`、`update_time`、`deleted` 为标准字段，由框架自动填充。

### 步骤 4：创建 Mapper

```java
package cn.iocoder.yudao.module.product.dal.mysql;

import cn.iocoder.yudao.framework.common.pojo.PageResult;
import cn.iocoder.yudao.framework.mybatis.core.mapper.BaseMapperX;
import cn.iocoder.yudao.framework.mybatis.core.query.LambdaQueryWrapperX;
import cn.iocoder.yudao.module.product.controller.admin.vo.ProductPageReqVO;
import cn.iocoder.yudao.module.product.dal.dataobject.ProductDO;
import org.apache.ibatis.annotations.Mapper;

@Mapper
public interface ProductMapper extends BaseMapperX<ProductDO> {

    default PageResult<ProductDO> selectPage(ProductPageReqVO reqVO) {
        return selectPage(reqVO, new LambdaQueryWrapperX<ProductDO>()
                .likeIfPresent(ProductDO::getName, reqVO.getName())
                .eqIfPresent(ProductDO::getStatus, reqVO.getStatus())
                .betweenIfPresent(ProductDO::getCreateTime, reqVO.getCreateTime())
                .orderByDesc(ProductDO::getId));
    }
}
```

```java
package cn.iocoder.yudao.module.product.service;

import cn.iocoder.yudao.framework.common.pojo.PageResult;
import cn.iocoder.yudao.framework.common.util.object.BeanUtils;
import cn.iocoder.yudao.module.product.controller.admin.vo.ProductPageReqVO;
import cn.iocoder.yudao.module.product.controller.admin.vo.ProductSaveReqVO;
import cn.iocoder.yudao.module.product.dal.dataobject.ProductDO;
import cn.iocoder.yudao.module.product.dal.mysql.ProductMapper;
import jakarta.annotation.Resource;
import org.springframework.stereotype.Service;
import org.springframework.validation.annotation.Validated;

import static cn.iocoder.yudao.framework.common.exception.util.ServiceExceptionUtil.exception;
import static cn.iocoder.yudao.module.product.enums.ErrorCodeConstants.PRODUCT_NOT_EXISTS;

@Service
@Validated
public class ProductServiceImpl implements ProductService {

    @Resource
    private ProductMapper productMapper;

    @Override
    public Long createProduct(ProductSaveReqVO createReqVO) {
        ProductDO product = BeanUtils.toBean(createReqVO, ProductDO.class);
        productMapper.insert(product);
        return product.getId();
    }

    @Override
    public void updateProduct(ProductSaveReqVO updateReqVO) {
        validateProductExists(updateReqVO.getId());
        ProductDO updateObj = BeanUtils.toBean(updateReqVO, ProductDO.class);
        productMapper.updateById(updateObj);
    }

    @Override
    public void deleteProduct(Long id) {
        validateProductExists(id);
        productMapper.deleteById(id);
    }

    @Override
    public ProductDO getProduct(Long id) {
        return productMapper.selectById(id);
    }

    @Override
    public PageResult<ProductDO> getProductPage(ProductPageReqVO pageReqVO) {
        return productMapper.selectPage(pageReqVO);
    }

    private void validateProductExists(Long id) {
        if (productMapper.selectById(id) == null) {
            throw exception(PRODUCT_NOT_EXISTS);
        }
    }
}
```

### 步骤 7：创建 Controller

```java
package cn.iocoder.yudao.module.product.controller.admin;

import cn.iocoder.yudao.framework.common.pojo.CommonResult;
import cn.iocoder.yudao.framework.common.pojo.PageResult;
import cn.iocoder.yudao.framework.common.util.object.BeanUtils;
import cn.iocoder.yudao.module.product.controller.admin.vo.*;
import cn.iocoder.yudao.module.product.dal.dataobject.ProductDO;
import cn.iocoder.yudao.module.product.service.ProductService;
import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.Parameter;
import io.swagger.v3.oas.annotations.tags.Tag;
import jakarta.annotation.Resource;
import jakarta.validation.Valid;
import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.validation.annotation.Validated;
import org.springframework.web.bind.annotation.*;

import static cn.iocoder.yudao.framework.common.pojo.CommonResult.success;

@Tag(name = "管理后台 - 商品管理")
@RestController
@RequestMapping("/product/product")
@Validated
public class ProductController {

    @Resource
    private ProductService productService;

    @PostMapping("/create")
    @Operation(summary = "创建商品")
    @PreAuthorize("@ss.hasPermission('product:product:create')")
    public CommonResult<Long> createProduct(@Valid @RequestBody ProductSaveReqVO createReqVO) {
        return success(productService.createProduct(createReqVO));
    }

    @PutMapping("/update")
    @Operation(summary = "更新商品")
    @PreAuthorize("@ss.hasPermission('product:product:update')")
    public CommonResult<Boolean> updateProduct(@Valid @RequestBody ProductSaveReqVO updateReqVO) {
        productService.updateProduct(updateReqVO);
        return success(true);
    }

    @DeleteMapping("/delete")
    @Operation(summary = "删除商品")
    @Parameter(name = "id", description = "编号", required = true)
    @PreAuthorize("@ss.hasPermission('product:product:delete')")
    public CommonResult<Boolean> deleteProduct(@RequestParam("id") Long id) {
        productService.deleteProduct(id);
        return success(true);
    }

    @GetMapping("/get")
    @Operation(summary = "获取商品详情")
    @Parameter(name = "id", description = "编号", required = true)
    @PreAuthorize("@ss.hasPermission('product:product:query')")
    public CommonResult<ProductRespVO> getProduct(@RequestParam("id") Long id) {
        ProductDO product = productService.getProduct(id);
        return success(BeanUtils.toBean(product, ProductRespVO.class));
    }

    @GetMapping("/page")
    @Operation(summary = "获取商品分页")
    @PreAuthorize("@ss.hasPermission('product:product:query')")
    public CommonResult<PageResult<ProductRespVO>> getProductPage(@Valid ProductPageReqVO pageReqVO) {
        PageResult<ProductDO> pageResult = productService.getProductPage(pageReqVO);
        return success(BeanUtils.toBean(pageResult, ProductRespVO.class));
    }
}
```

## 六、使用代码生成器（推荐）

项目内置了强大的代码生成器，可以基于数据库表自动生成 Controller、Service、Mapper、DO、VO 和前端页面。

### 6.1 使用步骤

1. **创建数据库表**（遵循字段规范，包含 `tenant_id`、`creator` 等标准字段）
2. **访问管理后台**：进入【基础设施 → 代码生成】
3. **导入表结构**：点击【导入】，选择需要生成代码的数据库表
4. **编辑生成配置**：
   - 表信息：模块名、业务名、类名前缀等
   - 字段信息：表单类型、查询方式、是否列表展示等
5. **预览代码**：确认生成的代码是否符合预期
6. **下载/生成代码**：下载 zip 包或直接生成到项目中

### 6.2 相关配置

在 `application.yaml` 中配置代码生成器：

```yaml
yudao:
  codegen:
    base-package: cn.iocoder.yudao
    db-schemas: ${spring.datasource.dynamic.datasource.master.name}
    front-type: 20    # 前端模板类型
    vo-type: 10       # VO 类型
```

## 十一、常见开发场景

### 11.1 新增不需要登录的接口

在模块的 `AuthorizeRequestsCustomizer` 实现类中配置：

```java
@Bean
public AuthorizeRequestsCustomizer authorizeRequestsCustomizer() {
    return new AuthorizeRequestsCustomizer() {
        @Override
        public void customize(AuthorizeHttpRequestsConfigurer.AuthorizationManagerRequestMatcherRegistry registry) {
            registry.requestMatchers("/admin-api/product/product/public/**").permitAll();
        }
    };
}
```

或在 `application.yaml` 中配置：

```yaml
yudao:
  security:
    permit-all_urls:
      - /admin-api/product/product/public/**
```
